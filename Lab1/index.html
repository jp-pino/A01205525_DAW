¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la red de redes, y como tal proporciona la estructura para conectar las computadoras alrededor del mundo. En cambio la World Wide Web es el modelo de acceso a información por medio del internet. Aunque estén relacionadas, el internet se refiere a la red física y la WWW al protocolo que se usa en ella.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Para enviar indormación a través de un formulario a un sitio web siempre es recomendable usar el método POST, más aun en casos de inicio de sesión puesto que de esta manera se pueden proteger las credenciales.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Indica que la petición fue exitosa. Su significado específico depende de del método empleado

  • GET

    El recurso ha sido captado y es transmitido en el cuerpo del mensaje.

  • HEAD

    Los headers de la entidad están en el cuerpo del mensaje.

  • POST

    EL resultado de la acción está en el cuerpo del mensaje.

  • TRACE

    El cuerpo del mensaje contiene la petición que recibió el servidor.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, porque el error 404 indica que no se ha podido encontrar el recurso en cuestión. Esto puede ser causado por vínculos rotos en la página que le impiden al usuario acceder a todos los recursos. Por esto, el desarrollador debe asegurarse de que todas las paginas estén correctamente vinculadas.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

No, porque el error 500 indica que el usuario no tiene los permisos requeridos para acceder al recurso solicitado. Esto va relacionado a la seguridad del sitio y que los usuarios no puedan editar los contenidos de la misma o acceder a recursos sin haberse autenticado.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Cuando una etiqueta de HTML es reemplazada en una nueva versión por etiquetas más funcionales o flexibles el W3C declara esa etiqueta como depreciada/desaprobada. Esto significa que a medida que se actualicen o se hagan nuevas páginas deberán implementar las nuevas etiquetas. Las etiquetas desaprobadas eventualmente dejarán de ser compatibles con los nuevos navegadores. Entre estas etiquetas encontramos: <acronym> <applet> <big> <basefont> <dir> <font> <frame>

Métodos

Descripción de métodos empleados en distintos elementos del desarrollo de aplicaciones web.

XML

Los métodos de XML son accesados a través de JavaScript mediante el DOM. Este permite encontrar los elementos de el documento XML por sus etiquetas.

XHTML

A través de la implementación en JavaScript del DOM de XHTML se pueden accessar/leer/editar los elementos en la página.

HTML5

De la misma manera que en los casos anteriores, los métodos definidos en el DOM de HTML5 permiten leer o modificar elementos de documento.

JS

Además de estar disponibles los métodos definidos por el DOM de HTML y XML, hay métodos de JavaScript con diferentes funciones de edición del texto/formato del documento, funciones matemáticas, de fecha y hora del sistema, ejecutar funciones del usuario, etc.

PHP

En PHP hay un gran número de métodos pensados para la creación de sitios dinámicos. Estos incluyen funciones matemáticas, de fecha/hora, de acceso a bases de datos, de manejo de arreglos, de seguridad, de estilo, etc.

JSP

JSP o JavaServer Pages es una tecnología para desarrollar páginas dinámicas con el lenguaje Java. De esta manera es similar a PHP, pero JSP tiene acceso a los métodos disponibles en Java.

AJAX

AJAX permite el intercambio de información con el servidor sin necesidad de volver a cargar toda la página. jQuery contiene métodos de AJAX para hacer enviar o pedir información del servidor implementando los métodos POST y GET de HTTP.

Errores

Ejemplos de códigos de errores de distintos elementos del desarrollo de aplicaciones web.

HTTP

400: BAD REQUEST

403: FORBIDDEN

404: NOT FOUND

500: INTERNAL SERVER ERROR

503: SERVICE UNAVAILABLE

XML

1: Error interno / general.

2: XML Inválido.

3: Cantidad inválida.

4: Violación de seguridad.

5: Los contenidos del elemento de orden son inválidos.

6: Una o más órdenes de un envío en batch son inválidas.

7: Los contenidos del elemento de pago son inválidos.

8: Servicio no disponible

XHTML

XHTML es HTML escrito como XML, siguiendo todas las reglas de HTML.

HTML5

Los errores en HTML5 se asocian a no cerrar apropiadamente las etiquetas abiertas o las comillas

JS

Uncaught ReferenceError: foo is not defined

Uncaught SyntaxError: Unexpected token o in JSON at position 1

Uncaught RangeError: Invalid array length

Uncaught TypeError: Cannot read property 'baz' of undefined

PHP

UPLOAD_ERR_OK: no hay error.

UPLOAD_ERR_INI_SIZE: el archivo subido excede el tamaño máximo especificado en "php.ini".

UPLOAD_ERR_FORM_SIZE: el archivo subido excede el tamaño máximo especificado en el formulario.

UPLOAD_ERR_PARTIAL: el archivo solo se subio parcialmente.

UPLOAD_ERR_NO_FILE: no se subió el archivo.

UPLOAD_ERR_NO_TMP_DIR: falta carpeta temporal.

UPLOAD_ERR_CANT_WRITE: no se pudo escribir al disco.

UPLOAD_ERR_EXTENSION: una extensión de PHP detuvo la carga del archivo.

JSP

Un servidor con JSP regresa errores del tipo HTTP como los mencionados anteriormente.

AJAX

Debido a que AJAX realiza solicitudes y envíos de información mediante el protocolo HTTP, los errores que puede regresar son aquellos que pertencen a HTTP y que fueron mencionados previamente.
